# Обзор главного меню

Главное меню - это точка входа в повседневое сопровождение сервера. Оно показывает операции верхнего уровня, а уже из них вы попадаете в подменю сайтов, расширений и безопасности.

## Структура верхнего уровня

Сейчас в главном меню доступны:

1. `List of sites dirs`
2. `Add/Change site`
3. `Add/Remove FTP user`
4. `Add/Change global PHP version`
5. `Settings SMTP sites`
6. `MySQL`
7. `PostgreSQL`
8. `Installing Extensions`
9. `Security settings`
10. `Change server timezone`
11. `Update server`

- `R` - перезагрузка сервера

Дополнительно есть сервисные команды:

- `update_menu` - обновление самого меню.

## Что меняется, а что нет

Полезно разделять пункты на три типа:

- **информационные** - только показывают состояние, например список сайтов;
- **конфигурационные** - меняют сайты, PHP, SMTP, БД, security;
- **системные** - обновляют пакеты, перезагружают или выключают сервер.

## Что особенно важно для production

С повышенной осторожностью используйте:

- удаление сайтов и БД;
- major-upgrade PostgreSQL и MySQL;
- изменение SSH-настроек;
- включение автообновлений и autorеboot;
- системные действия `Update server` и `R`.

## Как пользоваться документацией дальше

Навигация в левой панели повторяет структуру реального меню:

- для сайтов открыт отдельный вложенный раздел `2. Add/Change site`, внутри которого теперь находятся и выпуск сертификата, и переключение HTTP -> HTTPS;
- `Installing Extensions` и `Security settings` вынесены в отдельные подразделы;
- `MySQL` и `PostgreSQL` теперь доступны прямо на верхнем уровне главного меню;
- для остальных пунктов есть отдельные страницы верхнего уровня.
